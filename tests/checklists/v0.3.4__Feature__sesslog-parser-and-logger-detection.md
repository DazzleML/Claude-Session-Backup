# Human Test Checklist: sesslog_parser + logger detection (v0.3.4)

**Target version:** v0.3.4
**Ships with commit:** v0.3.4: feat: sesslog_parser + vault-aware default preference
**Companion automated tests:** 28 new (24 in `test_sesslog_parser.py`, 4 in `test_search.py`). Total 547/547 pass.

## What this is

Logger-side parity for the v0.3.x track. Three things to confirm by hand:

1. **`sesslog_parser` extracts file-ops from real logger output** -- format the logger writes to `.sesslog_*` / `.tools_*` / `.fileio_*` is correctly understood end-to-end.
2. **Sub-channel discovery finds the right files** -- `.sesslog_`, `.tools_`, `.fileio_` siblings of a session directory.
3. **Auto-mode dispatcher adapts to vault state** -- users with the logger see the full preference; users without see the collapsed `("fts5", "jsonl")`.

`csb search` user-facing surface is unchanged. v0.3.4 ships data extraction + smarter defaults; future commits (`csb files` / `csb search -d`) will consume the file-op data.

## Prerequisites

1. csb v0.3.4 installed (`pip install -e ".[dev]"`)
2. claude-session-logger has been writing to `~/.claude/sesslogs/` at some point (so `.sesslog_*` / `.tools_*` files exist on disk)
3. At least one session that did real work (Read/Edit/Write/Grep) so the parser has something to find

## High-Value Verification (~5 minutes)

### HV.1 -- Parser extracts file-ops from a real `.tools_*` file

- [ ] **Steps (PowerShell):**
  ```powershell
  $log = (Get-ChildItem ~/.claude/sesslogs/ -Recurse -Filter ".tools_*" | Select-Object -First 1).FullName
  python -c "
  from pathlib import Path
  from claude_session_backup.sesslog_parser import iter_file_ops_from_sesslog
  rows = list(iter_file_ops_from_sesslog(Path(r'$log'), 'test-sid'))
  from collections import Counter
  print(f'Total: {len(rows)}')
  for (op, s), n in sorted(Counter((r.operation, r.strength) for r in rows).items()):
      print(f'  {op:>16}  strength={s}  count={n}')
  "
  ```
- [ ] **Expected:** non-zero count; operations distributed across `read` (strength=2), `edited`/`wrote`/`notebook_edit` (strength=3), and possibly `searched` (strength=1).

### HV.2 -- Strength values match the JSONL walker

- [ ] **Steps:** verify the parser's strength assignment matches `transcript_walker._FILE_OP_TOOLS`:
  ```powershell
  python -c "
  from claude_session_backup.sesslog_parser import _FILE_OP_TOOLS as L
  from claude_session_backup.transcript_walker import _FILE_OP_TOOLS as W
  for tool in ('Read', 'Edit', 'Write', 'NotebookEdit', 'Grep'):
      ls = L.get(tool, (None, None, None))[1]
      ws = W.get(tool, (None, None, None))[2]
      match = '=' if ls == ws else '!='
      print(f'  {tool:>14}: logger={ls}  {match}  walker={ws}')
  "
  ```
- [ ] **Expected:** every line shows `=` (same strength for same op).

### HV.3 -- Sub-channel discovery finds session files

- [ ] **Steps:** pick a session directory and list its file-op channels:
  ```powershell
  $sessdir = (Get-ChildItem ~/.claude/sesslogs/ -Directory | Select-Object -First 1).FullName
  python -c "
  from pathlib import Path
  from claude_session_backup.sesslog_parser import find_fileop_channels_for_session
  paths = find_fileop_channels_for_session(Path(r'$sessdir'))
  for p in paths: print(' ', p.name)
  "
  ```
- [ ] **Expected:** zero or more files, all starting with `.sesslog_`, `.tools_`, or `.fileio_`. Files starting with `.convo_` (no file-op data) should NOT appear.

## Section 1: Parser format coverage

### 1.1 Read with line range stripped

- [ ] Construct a synthetic log line and parse it:
  ```powershell
  python -c "
  from pathlib import Path
  import tempfile
  from claude_session_backup.sesslog_parser import iter_file_ops_from_sesslog
  d = tempfile.mkdtemp()
  p = Path(d) / 'x.log'
  p.write_text('[[2026-05-18 10:00:00]] {Read: \"C:/code/foo.py:42\" }\n', encoding='utf-8')
  for r in iter_file_ops_from_sesslog(p, 'sid'):
      print(r.operation, r.file_path)
  "
  ```
- [ ] **Expected:** `read C:/code/foo.py` (line range stripped).

### 1.2 Grep with `in "<path>"` body

- [ ] Same harness, line:
  ```
  [[2026-05-18 10:00:00]] {Grep: pattern | "**/*.md" in "private/" }
  ```
- [ ] **Expected:** `searched private/`.

### 1.3 Agent-attributed tags

- [ ] Line: `[[2026-05-18 10:00:00]] {Read|Explore: "C:/code/foo.py" }`
- [ ] **Expected:** `read C:/code/foo.py` (the `|Explore` is parsed away).

### 1.4 Bash / Skill / Task / WebSearch are SKIPPED

- [ ] Lines:
  ```
  [[t]] {Bash: rm foo }
  [[t]] {Skill: commit }
  [[t]] {Agent: do x }
  [[t]] {WebSearch: query }
  ```
- [ ] **Expected:** zero file-op rows.
