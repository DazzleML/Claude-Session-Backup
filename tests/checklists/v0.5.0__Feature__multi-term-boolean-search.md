# Human Test Checklist: multi-term boolean `csb search` (v0.5.0)

**Target version:** v0.5.0
**Ships with commit:** v0.5.0: feat: multi-term boolean csb search (#49) + manual PyPI publish trigger
**Companion automated tests:** 3 new in `tests/test_search.py` (cross-message AND/OR/unordered, single-term-unchanged, operator-word-literal). Total 952 pass, 1 skipped.

## What this is

`csb search` now accepts multiple terms and combines them at the **session level**: `csb search "f-mv" "f-cp"` finds sessions containing **both** (default AND), in any order, even in different messages. `--match any` = OR. Operators live in `--match`, so a term like `AND` is literal. Mocks can't exercise a real multi-project vault, so this verifies by hand.

## Prerequisites

1. csb v0.5.0 installed (`pip install -e ".[dev]"`).
2. A real `~/.claude` vault with a session you know contains two specific terms (e.g. a tool-building session with `f-mv` and `f-cp`).

## High-Value Verification (~5 min)

### HV.1 — AND finds a session with both terms (cross-message)
- [ ] `csb search "f-mv" "f-cp" --only sessions` (pick two terms you know co-occur in one session).
- [ ] **Expected:** the session is listed. The terms may be in *different* messages — it still qualifies.

### HV.2 — AND excludes sessions missing a term
- [ ] `csb search "f-mv" "a-term-that-isnt-in-that-session" --only sessions`.
- [ ] **Expected:** the session is NOT listed (AND requires every term). Output names the terms + `(--match all)`.

### HV.3 — OR (`--match any`)
- [ ] `csb search "f-mv" "some-other-term" --match any --only sessions`.
- [ ] **Expected:** sessions containing *either* term are listed (superset of the AND result).

### HV.4 — Single term unchanged
- [ ] `csb search "f-mv" --only sessions` behaves exactly as before (same results).

### HV.5 — Operator word is literal
- [ ] `csb search "AND" --only sessions` (or another literal `AND`/`OR`) searches the literal word — it is NOT parsed as a boolean operator (operators are the `--match` flag).

### HV.6 — Drill-in hint round-trips
- [ ] In `--only sessions` output, the trailing `Next: csb search "f-mv" "f-cp" --session-id ...` line is copy-paste runnable.

### HV.7 — dir-scope rejection
- [ ] `csb search "f-mv" "f-cp" -d .` prints a clear error (multi-term not supported with `-d/-D` yet), not a wrong/partial result.

## Section 1: highlighting + modes
- [ ] Default (human) mode: in a multi-term search, **both** terms are highlighted (bold green) within the excerpts.
- [ ] `-A 3 -B 1` context, `--deleted all`, `--sort oldest`, `--limit N`, `--json` all still work with multiple terms.
