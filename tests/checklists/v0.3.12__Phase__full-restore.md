## What this is

A hand-runnable validation pass for v0.3.12's headline change: `csb restore <uuid>` now brings back the full SESSION-HISTORY footprint (main transcript + subagents + tool-results + remote-agents + claude-session-logger state files + sesslogs), not just the top-level JSONL.

The pre-v0.3.12 behavior (JSONL only) ships behind `--jsonl-only` for users who explicitly want just the transcript. A new `--force` flag opts into overwriting present on-disk files; by default, the policy is **preserve present, restore missing** -- safe, idempotent, never clobbers local content.

External testers: this checklist is self-contained -- you don't need to read the design docs to run it.
Project maintainers: see [Planning lineage](#planning-lineage) for design context.

## Prerequisites

1. **Windows 10/11 or Linux/macOS**.
2. **csb v0.3.12 installed in editable mode**:
   ```
   pip install -e .
   ```
3. **`preserve` tool installed** (for the empirical loop in Section 3). Available from the `preserve` package; `preserve --version` should print `Preserve v0.7.x` or later.
4. **GPG agent running** (Kleopatra on Windows) -- only matters if you intend to commit. Skip for read-only checklist passes.
5. **A scratch `~/.claude/`** -- either your real one (this checklist is safe; uses `preserve MOVE` for destructive sections so everything is recoverable) OR a throwaway clone in `%TEMP%` / `/tmp` if you want full isolation.

Always use `--db <path>` to point csb at a scratch DB rather than your real `~/.claude/session-backup.db` if you're working in a clone. The default DB path is `~/.claude/session-backup.db`.

## Section 3: Empirical restore-gap loop on a real session

This is the load-bearing test for v0.3.12. It moves a session's full on-disk footprint to a temp dir, runs `csb restore`, and confirms the new full-restore brings back the SESSION-HISTORY paths (not just the JSONL).

**SAFETY**: every destructive step uses `preserve MOVE` (copy-then-delete with SHA256 verification + auto-manifest). The final `preserve RESTORE` puts everything back byte-identical with mtime preserved. If anything looks wrong mid-loop, run `preserve RESTORE --src <temp-dir> --force` to roll back to the original state.

### 3a. Pick a real deleted session and capture state-A

- [ ] Pick a deleted session UUID from `csb list --deleted -n 5`. Call it `$UUID`. The session should have subagents (look for the [DELETED] tag plus a meaningful message count; a single-turn session won't have sidecars).
- [ ] Capture the on-disk footprint as state-A:
   ```bash
   find ~/.claude -name "*$UUID*" 2>/dev/null | sort | tee /tmp/state-A.txt | wc -l
   ```
   Expected: N paths where N >= 1. Note: `find -name` matches paths whose own name contains the UUID, so directories under `projects/<slug>/<uuid>/` (whose ancestors have the UUID but whose own names don't) will not appear -- they're still on disk, just not counted by this filter.

### 3b. Move the session's full footprint to a temp dir

- [ ] Enumerate every direct on-disk path matching the UUID. Build a `preserve MOVE` invocation that includes each. For a session with the typical 11-category footprint:
   ```bash
   preserve MOVE --abs --hash SHA256 --recursive --dst "/tmp/restore-test-$UUID" \
     --include ~/.claude/projects/<slug>/$UUID.jsonl \
     --include ~/.claude/session-states/$UUID.json \
     --include ~/.claude/session-states/$UUID.name-cache \
     --include ~/.claude/session-states/$UUID.source \
     --include ~/.claude/sesslogs/<dir-name-containing-uuid> \
     --include ~/.claude/debug/$UUID.txt \
     --include ~/.claude/session-env/$UUID \
     --include ~/.claude/tasks/$UUID \
     --include ~/.claude/todos/$UUID-agent-$UUID.json \
     --include ~/.claude/telemetry/1p_failed_events.$UUID.*.json \
     --include ~/.claude/file-history/$UUID
   ```
   Adjust the path list per what `find` actually shows. Skip any path that doesn't exist on your system (e.g. if you don't have `claude-session-logger`, the `session-states/` and `sesslogs/` paths won't exist).
- [ ] After preserve MOVE: clean up the empty directory leftovers (preserve doesn't rmdir parents). Use `rmdir` on each empty dir matching the UUID under `~/.claude/`.
- [ ] Confirm state-B is empty:
   ```bash
   find ~/.claude -name "*$UUID*" 2>/dev/null | wc -l
   ```
   Expected: `0`.

### 3c. Run `csb restore` and capture state-C

- [ ] `csb restore $UUID` (no flags -- default full restore). Output should report:
   - `Restored N files from commit XXXXXXXX.`
   - A category breakdown (main transcript / subagents / tool-results / remote-agents / session-states / sesslogs)
   - `Session should now be visible in Claude Code.`
- [ ] Capture state-C:
   ```bash
   find ~/.claude -name "*$UUID*" 2>/dev/null | sort
   ```
   Compare against state-A. Differences should ONLY be the ephemeral categories (`debug/`, `telemetry/`, `file-history/`, `tasks/`, `todos/`, `session-env/`, user-managed `sesslogs/bak/`) which are NOT restored by design.

### 3d. Idempotent re-run (zero-write verification)

- [ ] `csb restore $UUID` again. Output should report:
   - `Nothing to restore: all N expected files are already on disk. Use --force to overwrite from git history if you need to revert local changes.`
   - rc=0.
- [ ] **Strong "no sneaky writes" check** -- capture mtime+size of every restored file before AND after another no-op restore, then `diff`. Empty diff proves zero files were modified.
   ```bash
   UUID=$UUID  # set from earlier
   # BEFORE: capture mtime + size of every file under the SESSION-HISTORY scope
   {
     ls ~/.claude/projects/<slug>/$UUID.jsonl
     find ~/.claude/projects/<slug>/$UUID/ -type f 2>/dev/null
     ls ~/.claude/session-states/$UUID.* 2>/dev/null
     find ~/.claude/sesslogs/*__${UUID}_*/ -type f 2>/dev/null
   } | xargs -I{} stat -c "%Y %s %n" "{}" | sort > /tmp/before.txt

   # Run csb restore -- should be a no-op
   csb restore $UUID

   # AFTER: capture the same set
   {
     ls ~/.claude/projects/<slug>/$UUID.jsonl
     find ~/.claude/projects/<slug>/$UUID/ -type f 2>/dev/null
     ls ~/.claude/session-states/$UUID.* 2>/dev/null
     find ~/.claude/sesslogs/*__${UUID}_*/ -type f 2>/dev/null
   } | xargs -I{} stat -c "%Y %s %n" "{}" | sort > /tmp/after.txt

   # DIFF: must be empty (no file touched)
   diff /tmp/before.txt /tmp/after.txt && echo "IDENTICAL -- zero sneaky writes"
   ```
   Expected: `IDENTICAL -- zero sneaky writes`. If `diff` shows ANY line, the run modified a file -- investigate.
- [ ] Cross-check via `~/.claude/.session-backup.lock` -- should not exist either before or after (cleanly acquired + released by the context manager, not lingering).

### 3e. Preserve-present policy

- [ ] Modify a restored file slightly (e.g. append a line to the JSONL or a sesslog):
   ```bash
   echo '{"manual-edit":true}' >> ~/.claude/projects/<slug>/$UUID.jsonl
   ```
- [ ] `csb restore $UUID` again. Output should still report "Nothing to restore: all N expected files are already on disk." -- the file is present even though its bytes differ from git, so it's PRESERVED.
- [ ] Verify the manual edit survived:
   ```bash
   tail -1 ~/.claude/projects/<slug>/$UUID.jsonl
   ```
   The `{"manual-edit":true}` line must still be there. **csb did not clobber local content.**

### 3f. --force overrides preserve

- [ ] `csb restore $UUID --force`. Output should now report "Restored N files from commit XXXXXXXX."
- [ ] Verify the manual edit was overwritten:
   ```bash
   tail -1 ~/.claude/projects/<slug>/$UUID.jsonl
   ```
   The `{"manual-edit":true}` line should be gone -- git bytes won under `--force`.

### 3g. --jsonl-only flag

- [ ] Move one sidecar out of the way to simulate it being missing:
   ```bash
   mv ~/.claude/session-states/$UUID.json /tmp/$UUID.json.aside
   ```
- [ ] `csb restore $UUID --jsonl-only`. Output should report "Restored 1 file from commit XXXXXXXX." -- only the .jsonl, not the missing session-state.
- [ ] Verify the session-state file is STILL missing on disk -- csb restore --jsonl-only did not bring it back:
   ```bash
   ls ~/.claude/session-states/$UUID.json
   ```
   Expected: "No such file or directory".
- [ ] Restore it for the next section: `mv /tmp/$UUID.json.aside ~/.claude/session-states/$UUID.json`

### 3h. preserve RESTORE the ephemeral leftovers back

- [ ] `preserve RESTORE --src /tmp/restore-test-$UUID --force`. Reports "Restored M files, Skipped K" where K is the number of files csb restore put back (preserve sees they're identical and skips).
- [ ] Final state check:
   ```bash
   find ~/.claude -name "*$UUID*" 2>/dev/null | sort > /tmp/state-final.txt
   diff /tmp/state-A.txt /tmp/state-final.txt
   ```
   Expected: nothing missing from state-A. Extra entries are OK (`projects/<slug>/$UUID/` directory if csb restored subagents that weren't on disk in state-A).
- [ ] `preserve VERIFY --dst /tmp/restore-test-$UUID` confirms all SHA256 hashes match. Verified: N, Failed: 0, Missing: 0.

## Section 7: What mocks don't cover

Things that only a human running the real CLI can validate:

- **Cross-platform paths** -- the path translation (`_to_claude_dir_relative` etc.) handles `~/.claude/` being a subdir of the actual git repo root. Verify by running on a setup where `~/.claude/` is inside `~/.git`, AND on a setup where `~/.claude/` IS the git repo root.
- **Backup-lock contention** -- start `csb backup` in another shell mid-restore. The restore should hold `backup_lock` for the full multi-file write; the concurrent backup blocks until restore finishes.
- **GPG signing pass-through** -- the restore itself doesn't commit, but if you run `csb backup` after a restore, the noise/user commits should sign correctly (preserves the per-repo signing rule).
- **Rich-rendered output at narrow terminals** -- pipe `csb restore --dry-run` to `less -R`. The category breakdown lines should not overflow / wrap weirdly.
- **`claude --resume` end-to-end** -- the most important assertion. The restored session must actually load. Synthetic tests can't run Claude Code.
