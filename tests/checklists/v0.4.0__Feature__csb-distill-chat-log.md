# Human Test Checklist: csb distill -- Chat-Log Reading Layer

**Target version:** v0.4.0
**Ships with commit:** `v0.4.0: feat: csb distill -- read any session like a chat log (Closes #12)`
**Companion automated tests:** 20 in `tests/test_distill.py`; 904 total in the suite.

## What this is

`csb distill` renders a session as an instant-messenger-style chat log -- the alpha roadmap's final feature. The automated tests pin the format mechanically; this checklist covers what only eyes can judge: **does it actually read well** in Typora and Vim, on real multi-hundred-message sessions, with real agent identities and real tool noise filtered down to one-liners.

## Prerequisites

1. csb v0.4.0 (`csb --version` shows `ALPHA 0.4.0`).
2. A long real session (100+ messages, with agents if possible) -- find one via `csb list`.
3. Typora (or any Markdown viewer) and Vim/an editor for the dual-audience checks.
4. Cleanup notes at the end; canonical files land under `~/.claude/distilled/`.

## High-Value Verification (~5 minutes)

### HV.1 -- The reading experience (the entire point)

- [ ] **Steps:** `csb distill <big-session> -o`, open the printed path in **Typora**.
- [ ] **Expected:** reads like a chat log -- speaker turns visually obvious (generous blank separation), `YYYY/MM/DD hh:mm:ss <Identity>:` headers, message bodies rendering as Markdown (code blocks intact), tool calls as short indented one-liners you can skim past. You can follow the conversation WITHOUT scrolling walls of tool noise.
- [ ] **Why this matters:** "readable" is a human judgment; every mock passed and this could still feel wrong.

### HV.2 -- Agent identities

- [ ] **Steps:** pick a session that dispatched agents; distill; search the output for `<Agent`.
- [ ] **Expected:** agent turns appear as `<Agent:explore>` / `<Agent:senior-engineer>` (or bare `<Agent>`), clearly distinct from `<User>`/`<Claude>`.

### HV.3 -- Vim ergonomics

- [ ] **Steps:** open the distilled file in Vim; cursor onto a `[Read] C:/path/file.py` line; `gF` (or `gf`).
- [ ] **Expected:** Vim jumps to the file. Plain-text reading is comfortable (no markdown soup in the structural frame).

### HV.4 -- The three filters tell three stories

- [ ] **Steps:** `csb distill <s> --stdout --filter convo | head -50`, `--stdout --filter tools | head -30`, default both (reads the written file).
- [ ] **Expected:** convo = pure conversation; tools = a skimmable "what was done where" timeline under timestamped headers; both = conversation with tool one-liners attached beneath the message that did them.

### HV.5 -- Policy gate + backup integration

- [ ] **Steps** (cross-shell, csb only):
  ```
  csb config distill_policy never
  csb distill <s>                       # must refuse with the config hint
  csb config distill_policy always
  csb backup --no-commit                # regenerates stale canonical files
  csb config distill_policy on-demand   # restore the default
  ```
- [ ] **Expected:** `never` refuses even the explicit command; `always` makes backup report `Distilled N session(s) -> ...`; second backup run distills 0 (mtime-gated). Reset leaves you at the default.

## Section 1: Channels and edge cases

### 1.1 jsonl is the preferred channel (the verbatim record)

- [ ] **Steps:** distill any session whose transcript is on disk -- logger-era or not; check the header comment.
- [ ] **Expected:** `source: jsonl` (jsonl > sesslog > convo: distill renders csb's own preserved record); slash-command plumbing (`<command-name>` etc.) absent from the output.

### 1.2 Logger channels serve when the JSONL is gone

- [ ] **Steps:** find a session whose JSONL was purged but whose sesslogs dir survived (`csb list --deleted` candidates); distill it WITHOUT restoring (`--no-restore-pruned` won't apply if it's not DB-flagged; otherwise use `--source sesslog` on a live session to simulate); check header.
- [ ] **Expected:** `source: sesslog` (or `convo`); the conversation still renders from the logger's channels.

### 1.3 Stdout composability

- [ ] **Steps:** `csb distill <s> --stdout | head -40` in each shell (cmd.exe pipes through `more`, PowerShell `Select-Object -First 40`, POSIX `head`).
- [ ] **Expected:** clean streaming output; no codepage crashes on Unicode-heavy sessions (degrades to replacement chars at worst on cp1252 consoles).

### 1.4 Pruned session

- [ ] **Steps:** `csb distill <deleted-session> --no-restore-pruned` then with `--restore-pruned`.
- [ ] **Expected:** refuse-with-hint (says "not distilling"... wording mentions distill) / restore-then-render respectively.

## What the automated tests DON'T cover

- Whether it actually READS well (Typora/Vim feel, real session scale)
- Real agent-heavy sessions' identity rendering
- Console codepage behavior on real cp1252 terminals
- `gF` jump ergonomics
- POSIX end-to-end (#35)

## Reporting issues found

Note step, capture command + output (screenshot for readability judgments), file an issue referencing this checklist and #12.

## Cleanup

- `~/.claude/distilled/` files generated during testing: keep (they're the product) or remove the directory (prefer `dz safedel`); regeneration is one command away.
- `csb config distill_policy on-demand` if you changed it.
