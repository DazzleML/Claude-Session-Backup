# Human Test Checklist: ClaudePaths Layout Centralization

**Target version:** v0.4.3
**Ships with commit:** `v0.4.3: refactor: ClaudePaths -- one owner for the ~/.claude layout (Refs #46)`
**Companion automated tests:** 20 new in `tests/test_pathkit_layout.py` (+ nested-repo fixture in conftest); 933 total in the suite.

## What this is

A claimed-pure refactor -- and that's exactly why it gets a checklist (refactors touching file I/O, git pathspecs, and path conversion are where "no user-visible change" claims go to die). All `~/.claude` layout knowledge moved into one frozen `ClaudePaths` class; git pathspec/match strings now compose from the same name constants; three real defects found during the consolidation are fixed (hook relocation blind spot, junction `ValueError`, Windows test-path drift). The DB and git-history conventions must be BYTE-IDENTICAL to v0.4.2.

## Prerequisites

1. csb v0.4.3 (`csb --version`).
2. A scratch DB path for every live command -- NEVER the real index: `--db %TEMP%\csb-043-check.db`.

---

## Section 1 -- Output identical to v0.4.2 on real data (~5 min)

The refactor's core promise. With the scratch DB:

- [ ] `csb --db <scratch> backup --no-commit` -- scans all sessions, no errors, counts plausible
- [ ] `csb --db <scratch> list -n 5` -- timeline renders exactly as v0.4.2 did (names, folders, purge countdown)
- [ ] `cd` into a real project; `csb --db <scratch> scan .` -- folder-usage hits look right
- [ ] `csb --db <scratch> update backfill-deleted` -- enumerates deleted JSONLs from git history (this drives the composed `:(glob)projects/*/*.jsonl` pathspec AND both directions of repo-prefix translation on a home-rooted repo)
- [ ] `csb --db <scratch> distill <any-session-fragment>` -- writes the canonical `distilled/<slug>/<uuid>.md`

## Section 2 -- Junction/symlink claude_dir (~3 min, the RF2 fix)

- [ ] Make a junction to a COPY of a few sessions (never the real dir):
  `powershell -Command "New-Item -ItemType Junction -Path C:\tmp\claude-link -Target C:\tmp\claude-copy"`
- [ ] `csb --claude-dir C:\tmp\claude-link --db %TEMP%\junc.db backup --no-commit` -- completes WITHOUT `ValueError` (pre-#46 this path could raise on relativize)
- [ ] `csb --claude-dir C:\tmp\claude-link --db %TEMP%\junc.db list` -- sessions listed; `jsonl_path` values in the DB are forward-slash `projects/...` rows

## Section 3 -- Hook relocation island (~2 min)

- [ ] `set CLAUDE_CONFIG_DIR=C:\tmp\reloc` then run the hook directly: `python hooks/scripts/backup-hook.py < NUL` (or echo `{}` into it)
- [ ] Hook log appears under `C:\tmp\reloc\csb-logs\`, NOT under `~/.claude/csb-logs\`
- [ ] Unset the env var; rerun; log lands in `~/.claude/csb-logs\` as before

## Section 4 -- Guard rails hold (~1 min)

- [ ] `python -m pytest tests/test_pathkit_layout.py -q` -- 20 pass
- [ ] Optional spite-check: add `x = Path(claude_dir) / "projects"` to any module under `claude_session_backup/`, rerun -- the guard test names the file:line; revert

## What the automated tests DON'T cover

- Real-data output comparison against v0.4.2 by eyeball (Section 1)
- A junction pointing at a real-shaped session corpus (Section 2 -- the suite uses a minimal fixture)
- The hook running as an actual subprocess with env relocation (Section 3 -- the suite imports it)
- POSIX symlink claude_dir on real Linux/macOS (#35 rides along)
