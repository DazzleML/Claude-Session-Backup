# Human Test Checklist: csb search — FTS5 escaping + zero-match fallback (v0.4.7)

**Target version:** v0.4.7
**Ships with commit:** v0.4.7: fix: csb search — punctuated-term escaping + auto-fallback to source of truth
**Companion automated tests:** `escape_fts_query` hyphen unit test + 3 dispatch tests (`test_fts5_db.py`, `test_search.py`), all red-green verified. Total 949 pass, 1 skipped.

## What this is

Two bugs made `csb search` return silent **false negatives**:
- **Bug A:** `escape_fts_query("f-mv")` produced `"fmv"` (hyphen deleted, tokens joined) → FTS5 matched nothing. Now → `"f mv"` (adjacency phrase).
- **Bug B:** the auto dispatcher trusted a zero from the first fresh source with no fallback. Now auto-search **falls through to the authoritative JSONL** when the chosen source yields nothing (explicit `--source X` unchanged).

Mocks can't exercise a real multi-project vault with real FTS5 indices, so this verifies end-to-end by hand.

## Prerequisites

1. csb v0.4.7 installed (`pip install -e ".[dev]"`).
2. A real `~/.claude` vault with at least one FTS5-indexed session whose transcript contains a **hyphenated** term (e.g. a tool name like `f-mv`, or `claude-code`).

---

## High-Value Verification (~5 min)

### HV.1 — The reported bug is fixed
- [ ] Find a hyphenated term you know appears in an FTS5-indexed session (`csb scan` shows the session; the term is in its transcript). Run `csb search "<the-hyphenated-term>"`.
- [ ] **Expected:** the session is returned. **Before v0.4.7** it was silently omitted if that session had a fresh FTS5 index.

### HV.2 — Escaping produces a phrase, not a joined token
- [ ] `python -c "from claude_session_backup.fts5_db import escape_fts_query as e; print(e('f-mv'), '|', e('claude-code'), '|', e('oauth callback'))"`
- [ ] **Expected:** `"f mv" | "claude code" | "oauth" "callback"` — hyphens become spaces inside a phrase; multi-word input stays two AND'd terms.

### HV.3 — Auto fallback to the source of truth
- [ ] Pick a term that exists in a session's JSONL but that its FTS5 index might miss (e.g. text only in a tool/bash block). Run `csb search "<term>"` (auto) — it should be found.
- [ ] **Why it matters:** a fresh-but-deficient FTS5 must never shadow the authoritative transcript ("fresh ≠ complete").

### HV.4 — Explicit --source keeps the no-fallback contract
- [ ] For a term whose FTS5 returns nothing, `csb search "<term>" --source fts5` should still return nothing (you pinned that channel), while plain `csb search "<term>"` finds it via fallback.

### HV.5 — No perf cliff
- [ ] A broad search for a **rare** term (matches few/no sessions) should still complete in reasonable time. The fallback adds work only for zero-hit sessions; flag it if a broad search feels markedly slower than before.
