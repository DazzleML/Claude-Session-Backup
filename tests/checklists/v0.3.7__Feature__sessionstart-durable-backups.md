# Human Test Checklist: SessionStart durable backups (v0.3.7)

**Target version:** v0.3.7
**Ships with commit:** v0.3.7: feat: SessionStart backup hook + background execution
**Companion automated tests:** 19 in `tests/test_backup_hook.py` + updated `tests/test_hook.py`. Total 621/621 pass.

## What this is

csb now backs up on **SessionStart** (in addition to PreCompact + SessionEnd) and runs every hook-triggered backup **in the background**. This makes the just-ended / just-forked session reliably indexed without blocking the session. Mocks can't fire real Claude Code hooks or exercise process teardown, so this checklist verifies the end-to-end behavior by hand.

## Prerequisites

1. csb v0.3.7 installed (`pip install -e ".[dev]"`).
2. **Deploy the new hook registration:** the SessionStart hook is new in `hooks.json`, so Claude Code must re-read it.
   - PowerShell: `claude plugin update claude-session-backup@dazzle-claude-session-backup` then restart Claude Code.
   - (Directory-source marketplace runs the repo in-place, but the hook *registration* still needs the update + restart to take effect.)
3. A `~/.claude` git repo with sessions so `csb backup` has work to do.

## High-Value Verification (~5 min)

### HV.1 -- No startup pause (background execution)
- [ ] Start a Claude Code session. **Expected:** the prompt is interactive immediately -- no ~15-25s freeze while a full backup runs. (Backup runs in the background.)
- [ ] **Why it matters:** a synchronous full backup at every start would be a constant UX tax; background execution is the whole point.

### HV.2 -- SessionStart actually fired and ran
- [ ] After starting a session, wait ~30s, then (PowerShell):
  ```powershell
  Get-Content "$env:USERPROFILE\.claude\csb-logs\backup-SessionStart.log" -Tail 5
  ```
- [ ] **Expected:** a recent `start background backup: event=SessionStart source=startup` (or `resume`) line.

### HV.3 -- Start catch-up (the core promise)
- [ ] In session A, do something distinctive (say a unique phrase). Exit A. Start session B. Wait ~30s.
- [ ] `csb show <A-uuid>` resolves and `csb search "<unique phrase>"` finds it.
- [ ] **Why it matters:** even if A's SessionEnd backup was killed by teardown, B's SessionStart catches A up. "Nothing lost."

### HV.4 -- Fork coverage (the original bug)
- [ ] In a session, run `/rewind` (or `/fork`) and continue into the fork. Wait ~30s.
- [ ] `Get-Content "$env:USERPROFILE\.claude\csb-logs\backup-SessionStart.log" -Tail 5` shows a run with **`source=resume`**.
- [ ] `csb show <fork-uuid>` resolves; `csb search "<phrase typed in the fork>"` finds it.
- [ ] **Why it matters:** this is the exact `/rewind` scenario that was unsearchable before v0.3.7.

### HV.5 -- Compaction is not double-backed-up
- [ ] Trigger a compaction (or wait for auto-compaction). After it completes:
  ```powershell
  Get-Content "$env:USERPROFILE\.claude\csb-logs\backup-PreCompact.log" -Tail 3
  Get-Content "$env:USERPROFILE\.claude\csb-logs\backup-SessionStart.log" -Tail 3
  ```
- [ ] **Expected:** PreCompact shows a `start background backup`; the post-compaction SessionStart shows a **`skip: ... source=compact`** line (no redundant backup).

### HV.6 -- Lock stays healthy
- [ ] After several session starts/exits: `Test-Path "$env:USERPROFILE\.claude\.csb-backup.lock"` -- if present, confirm the next backup reclaims it (look for `reclaimed stale backup lock` in a `csb-logs\backup-*.log`), not a permanent block.

## Section 1: skip / run decision (no real backup)

### 1.1 compact skips (cross-shell)
- [ ] **PowerShell:**
  ```powershell
  '{"hook_event_name":"SessionStart","source":"compact"}' | python hooks\scripts\backup-hook.py
  Get-Content "$env:USERPROFILE\.claude\csb-logs\backup-SessionStart.log" -Tail 1
  ```
  **POSIX:**
  ```bash
  echo '{"hook_event_name":"SessionStart","source":"compact"}' | python hooks/scripts/backup-hook.py
  tail -1 ~/.claude/csb-logs/backup-SessionStart.log
  ```
- [ ] **Expected:** a `skip: SessionStart source=compact` line; **no** `.csb-backup.lock` created by this.

### 1.2 manual invocation runs (triggers a real background backup)
- [ ] `python hooks/scripts/backup-hook.py < /dev/null` (POSIX) / `$null | python hooks\scripts\backup-hook.py` (PowerShell).
- [ ] **Expected:** returns immediately; `backup-manual.log` shows `start background backup`; a real backup runs in the background. (Side effect: a real `csb backup` of `~/.claude`.)

## What the automated tests DON'T cover

- Real Claude Code hook firing (SessionStart/PreCompact/SessionEnd) and the `source` values it actually sends -- HV section.
- Process teardown killing a SessionEnd backup (the reason for SessionStart catch-up).
- Background backup *completing* after the hook returns (mocks stop at Popen).
- The `claude plugin update` + restart needed to register the new hook.
