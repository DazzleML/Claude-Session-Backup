# Human Test Checklist: csb view -- Session Viewer Launcher

**Target version:** v0.3.20
**Ships with commit:** `v0.3.20: feat: csb view -- open sessions in Claude Code History Viewer (Closes #14, #34)`
**Companion automated tests:** 23 in `tests/test_view.py` + 1 resume-failure test in `tests/test_restore.py`; 874 total in the suite.

## What this is

`csb view <anything-that-identifies-a-session>` resolves the session and opens it in Claude Code History Viewer (CCHV), detached. The automated tests mock the launch entirely -- this checklist covers what they can't: the real GUI opening focused on the right session, detachment semantics (viewer outlives the shell), dev-mode launches, and the pruned restore-then-view prompt on a real TTY.

## Prerequisites

1. csb v0.3.20 (`csb --version` shows `ALPHA 0.3.20`).
2. CCHV available for the launch tests. A standard install (`C:\Program Files\CCHistoryViewer\`) is auto-discovered -- zero config needed; skip to step 3. Otherwise point csb at a binary or dev repo (pick one):

   **cmd.exe**:
   ```cmd
   set CLAUDEVIEW_BIN=C:\path\to\claude-code-history-viewer.exe
   ```
   **PowerShell**:
   ```powershell
   $env:CLAUDEVIEW_BIN = "C:\path\to\claude-code-history-viewer.exe"
   ```
   **POSIX**:
   ```bash
   export CLAUDEVIEW_BIN=/path/to/claude-code-history-viewer
   ```
   Or persistently (cross-shell): `csb config viewer_path "<path>"`
3. Sections 1-2 are read-only. Section 3 (pruned) restores a session -- pick an expendable deleted one.
4. Reset: unset `CLAUDEVIEW_BIN` / `csb config viewer_path ""` afterward if you don't want it sticky.

## High-Value Verification (~5 minutes)

### HV.1 -- UUID prefix opens the right session in the GUI

- [ ] **Steps:** `csb view <8-char-prefix-of-a-known-session>`
- [ ] **Expected:** `Opening: <name> (<uuid8>...)` + path printed; CCHV opens **focused on that session** (not just the app home screen); the conversation matches the session name.
- [ ] **Why this matters:** end-to-end resolve->launch->focus is the entire feature; mocks verify the UUID handoff, only eyes verify the focus.

### HV.2 -- The viewer outlives the shell (detachment)

- [ ] **Steps:** run HV.1, then close the terminal window (or `exit`).
- [ ] **Expected:** CCHV stays open and responsive.
- [ ] **Why this matters:** the detached-Popen flags are platform-specific and untestable by mocks.

### HV.3 -- Keyword resolution + multi-match listing

- [ ] **Steps:** `csb view <keyword-matching-several-sessions>` then `csb view <keyword-matching-exactly-one>`
- [ ] **Expected:** multi-match prints the timeline of candidates + "Re-run with a UUID prefix" and exits 1 (no GUI); unique match opens the GUI with `Resolved via: search` shown.

### HV.4 -- No-viewer fallback is genuinely useful

- [ ] **Steps** (pick your shell to clear the override):

  **cmd.exe**: `set CLAUDEVIEW_BIN=` then `csb view <uuid-prefix>`
  **PowerShell**: `Remove-Item Env:\CLAUDEVIEW_BIN` then `csb view <uuid-prefix>`
  **POSIX**: `unset CLAUDEVIEW_BIN` then `csb view <uuid-prefix>`

  (Also ensure `viewer_path` config is unset and no installed copy exists, else skip.)
- [ ] **Expected:** session resolved, transcript path printed, three setup pointers shown, exit 0 -- reads as a helpful answer, not an error.

### HV.5 -- Pruned session: prompt, restore, view (real TTY)

- [ ] **Steps:** `csb view <prefix-of-a-deleted-session>` in an interactive terminal; answer `Y`.
- [ ] **Expected:** `Session '<name>' is pruned ... Restore from git before viewing? [Y/n]`; on Y: restore summary (files + symlinks + `Applied original timestamps`), then the GUI opens on the restored session. Answering `n` aborts with exit 0 and no restore.
- [ ] **Why this matters:** the #34 view-half, with v0.3.18 fidelity riding along -- prompt rendering and flow feel need eyes.

## Section 1: Resolution surfaces (read-only)

### 1.1 Directory and "."

- [ ] **Steps:** `cd` into a project folder with known sessions; `csb view .`
- [ ] **Expected:** unique session -> opens; several -> candidate timeline, exit 1.

### 1.2 Absolute .jsonl path

- [ ] **Steps:** `csb view "<absolute path to a projects/<slug>/<uuid>.jsonl>"`
- [ ] **Expected:** opens that exact session (`Resolved via: path`).

### 1.3 Sesslog folder name

- [ ] **Steps:** `csb view <NAME__...__uuid_host dir name from ~/.claude/sesslogs>`
- [ ] **Expected:** opens the embedded-UUID session (`Resolved via: sesslog-name`).

### 1.4 No-arg listing

- [ ] **Steps:** `csb view`
- [ ] **Expected:** recent-sessions timeline + usage hint; exit 0.

## Section 2: Discovery order and dev mode

### 2.1 Config key beats platform discovery; env beats config

- [ ] **Steps:** set `viewer_path` to one copy and `CLAUDEVIEW_BIN` to another; run; then unset env and re-run.
- [ ] **Expected:** env copy launches first; config copy after env is cleared.

### 2.2 Dev-mode launch (only if you have the CCHV repo)

- [ ] **Steps:** `csb config viewer_path "<path-to-cchv-repo>"` (a dir with package.json + src-tauri), then `csb view <prefix>`
- [ ] **Expected:** `Launching in dev mode from: ...` + foreground Vite/cargo output; Ctrl-C stops it cleanly.

## Section 3: Pruned flags (non-TTY)

### 3.1 --no-restore-pruned

- [ ] **Steps:** `csb view <deleted-prefix> --no-restore-pruned`
- [ ] **Expected:** refuses with the `csb view <uuid>` retry hint (says "viewing", not "resuming"); exit 1; nothing restored.

### 3.2 --restore-pruned non-interactive

- [ ] **Steps:** `csb view <deleted-prefix> --restore-pruned` (e.g. piped: `echo | csb view ...`)
- [ ] **Expected:** restores without prompting, then launches (or prints path if no viewer).

## What the automated tests DON'T cover

- The real GUI opening, focusing on the right session, and rendering it
- Detachment (viewer survives shell exit) on each platform
- Dev-mode pnpm/cargo launches
- TTY prompt rendering and interactive feel
- Discovery against real install locations (LOCALAPPDATA, /Applications, ~/.local/bin)
- POSIX end-to-end (#35 tracks the Linux walk)

## Reporting issues found

Note section + step, capture command + output (and a screenshot for GUI-focus failures), file an issue referencing this checklist and #14.

## Cleanup

- Unset `CLAUDEVIEW_BIN`; reset `viewer_path` (`csb config viewer_path ""`) if you set it for testing.
- Any session restored in HV.5 / Section 3: the files came from git -- remove the restored paths (prefer `dz safedel`) to return it to deleted state, or keep it.
