# Human Test Checklist: restore-by-name + CLAUDE_CONFIG_DIR Relocation

**Target version:** v0.4.2
**Ships with commit:** `v0.4.2: feat: restore-by-name + follow relocated ~/.claude (Closes #44, #45)`
**Companion automated tests:** 9 new (3 restore-by-name in `tests/test_restore.py`, 6 relocation in `tests/test_config.py`); 913 total in the suite.

## What this is

Two small features that round out real-world usage: `csb restore` accepts session names (the last command to adopt the shared resolver), and csb automatically follows `CLAUDE_CONFIG_DIR` -- Claude Code's own directory-relocation mechanism used by container / host-mount / worktree-isolated setups. This checklist covers the on-real-data feel and the relocation matrix on an actual moved directory.

## High-Value Verification (~4 minutes)

### HV.1 -- Restore by name (the README workflow, literally)

- [ ] **Steps:** `csb list --deleted`, copy a deleted session's NAME, then `csb restore <that-name>`
- [ ] **Expected:** resolves and restores -- no "supply the full UUID" detour. (On a LIVE session's name, the preserve-default makes it a harmless "Nothing to restore" no-op -- also a valid check.)
- [ ] **Why this matters:** you remember sessions by title from the list output; this closes the last name-blind command.

### HV.2 -- Ambiguous name lists candidates

- [ ] **Steps:** `csb restore <keyword matching several sessions>`
- [ ] **Expected:** "N sessions match" + candidates timeline, exit 1, nothing restored.

### HV.3 -- Relocation: CLAUDE_CONFIG_DIR alone is enough

- [ ] **Steps** (pick your shell):

  **cmd.exe**: `set CLAUDE_CONFIG_DIR=D:\moved-claude` then `csb list`
  **PowerShell**: `$env:CLAUDE_CONFIG_DIR = "D:\moved-claude"` then `csb list`
  **POSIX**: `CLAUDE_CONFIG_DIR=/srv/claude csb list`
- [ ] **Expected:** csb operates entirely on the relocated dir (sessions listed from there; `csb backup --no-commit` indexes there; the DB file appears INSIDE the relocated dir, not in `~/.claude`). Unset afterward.
- [ ] **Why this matters:** zero-config support for the container/worktree setups that relocate via Claude Code's own mechanism.

### HV.4 -- Precedence sanity

- [ ] **Steps:** with `CLAUDE_CONFIG_DIR` set, also pass `--claude-dir <other>` on one command.
- [ ] **Expected:** the flag wins (flag > CLAUDE_DIR > CLAUDE_CONFIG_DIR > config > ~/.claude).
