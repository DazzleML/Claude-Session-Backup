# Human Test Checklist: scan --deleted Grammar Unification

**Target version:** v0.3.19
**Ships with commit:** `v0.3.19: fix: csb scan adopts the --deleted [only|all] grammar (Closes #41)`
**Companion automated tests:** 9 new in `tests/test_cli.py` (grammar, normalizer, alias deprecation); 850 total in the suite.

## What this is

`csb scan` migrates from its legacy boolean `--deleted` + separate `--all` flags to the canonical two-valued `--deleted [only|all]` grammar that `list` and `search` already use (#41). This checklist verifies the human-facing parts: help-text rendering, the deprecation warning on real stderr, and that documented muscle-memory invocations still behave.

## Prerequisites

1. csb v0.3.19 installed (`csb --version` shows `ALPHA 0.3.19`).
2. At least one deleted and one live session in the index (`csb list --deleted -n 3` / `csb list -n 3`).
3. All commands below are read-only against the index -- no reset needed between steps. They are `csb`-only invocations, cross-shell safe as written.

## High-Value Verification (~3 minutes)

### HV.1 -- The three canonical forms all work on scan

- [ ] **Steps** (cross-shell, csb only):
  ```
  csb scan -d <some-project-path> --deleted
  csb scan -d <some-project-path> --deleted only
  csb scan -d <some-project-path> --deleted all
  ```
- [ ] **Expected:** first two show deleted-only for that scope (identical output); third shows live AND deleted. No argparse errors.
- [ ] **Why this matters:** `--deleted only` was a hard parse error before v0.3.19 -- this IS the fix.

### HV.2 -- Deprecated --all still works and warns exactly once

- [ ] **Steps:**
  ```
  csb scan -d <some-project-path> --all
  ```
- [ ] **Expected:** same output as `--deleted all`, PLUS one stderr line: `Warning: --all is deprecated; use --deleted all (removal planned for 0.4).` The warning appears once, before the results, and does not corrupt the table layout.
- [ ] **Why this matters:** silent breakage of documented muscle memory is the failure mode this guards.

### HV.3 -- --restore still implies deleted scope

- [ ] **Steps:**
  ```
  csb scan -d <path-with-a-deleted-session> --restore --dry-run
  ```
- [ ] **Expected:** previews ONLY deleted sessions' files (active sessions skipped), exactly as in v0.3.18 -- no `--deleted` flag needed.
- [ ] **Why this matters:** the implies-deleted precedence is load-bearing for the documented bulk-recovery flow.

### HV.4 -- list/search unchanged (regression)

- [ ] **Steps:**
  ```
  csb list --deleted
  csb list
  csb search <known-term> --deleted all
  ```
- [ ] **Expected:** `list --deleted` shows deleted-only; bare `csb list` still prints the "N deleted sessions hidden" footer when applicable (this footer's gate was touched in the refactor); search with `--deleted all` spans live + deleted.

## Section 1: Help and errors

### 1.1 Help text renders the shared grammar

- [ ] **Steps:** `csb scan --help`, `csb list --help`, `csb search --help`
- [ ] **Expected:** all three show the same `--deleted [{only,all}]` wording (verb varies: scan/show/search). `--all` does NOT appear in scan's help (hidden alias). No codepage artifacts on Windows cmd.exe.

### 1.2 Invalid choice gives a sane error

- [ ] **Steps:** `csb scan --deleted bogus`
- [ ] **Expected:** argparse error naming the valid choices (`'only', 'all'`); exit code 2; no traceback.

### 1.3 Token-order caveat behaves as documented

- [ ] **Steps:** `csb scan --deleted someterm` (a term that is not only/all)
- [ ] **Expected:** parse error (the documented caveat); `csb scan someterm --deleted` works.
