# Human Test Checklist: csb resume by Session Name (Shared Resolver)

**Target version:** v0.3.21
**Ships with commit:** `v0.3.21: feat: csb resume accepts session names + every csb view identifier (Closes #42)`
**Companion automated tests:** 6 new in `tests/test_view.py` (#42 section); 880 total in the suite.

## What this is

`csb resume` now accepts every identifier `csb view` accepts -- most importantly the **exact session name** (`csb resume MAKING-LIBS_...`), matching and exceeding `claude --resume`'s native surface. csb resolves the name to the UUID via its index and always hands Claude the full UUID. The automated tests intercept the `claude` launch -- this checklist covers the real thing: an actual `claude --resume` landing in the right conversation, from the right directory.

## Prerequisites

1. csb v0.3.21 (`csb --version` shows `ALPHA 0.3.21`).
2. A live (non-pruned) session whose NAME you know (`csb list` shows names). Resume launches a real interactive `claude` -- be ready to exit it (Ctrl-C / `/exit`).
3. All steps are read-only on csb's side; reset is just exiting the launched claude.

## High-Value Verification (~5 minutes)

### HV.1 -- Resume by exact session name (the headline)

- [ ] **Steps:** copy a full session name from `csb list`, then:
  ```
  csb resume <exact-session-name>
  ```
- [ ] **Expected:** csb resolves it (no "No session found"), cd-targets the session's start folder as usual, and `claude --resume <full-uuid>` opens THAT conversation. Exit claude afterward.
- [ ] **Why this matters:** this is #42 -- the wrapper now accepts at least everything the wrapped tool accepts.

### HV.2 -- Keyword multi-match lists candidates, never launches

- [ ] **Steps:** `csb resume <keyword matching several sessions>` (e.g. a project word)
- [ ] **Expected:** "N sessions match '<keyword>' (via search)" + the candidates timeline + exit 1. NO claude launched.

### HV.3 -- UUID and prefix still work exactly as before

- [ ] **Steps:** `csb resume <full-uuid>` and `csb resume <8-char-prefix>` on a known session.
- [ ] **Expected:** identical behavior to v0.3.20 -- direct resume; an ambiguous prefix still errors with the candidates list and exit code 2.

### HV.4 -- Parity sanity vs Claude native

- [ ] **Steps:** exit csb's launched session; run Claude natively: `claude --resume <same-exact-session-name>` from the session's project dir.
- [ ] **Expected:** Claude resolves the same session (if your Claude build has custom-title resume enabled) -- confirming the name csb matched is the same `customTitle` Claude matches. If Claude instead opens the picker, that's its feature-gate -- csb's resolution working anyway is the point.

## Section 1: Other identifier forms on resume

### 1.1 Directory / "."

- [ ] **Steps:** `cd` into a project folder; `csb resume .`
- [ ] **Expected:** unique session -> resumes it; several -> candidates, exit 1.

### 1.2 Sesslog folder name

- [ ] **Steps:** `csb resume <NAME__...__uuid_host>` (a dir name from `~/.claude/sesslogs/`)
- [ ] **Expected:** resumes the embedded-UUID session.

### 1.3 Pruned session by name

- [ ] **Steps:** `csb resume <name-of-a-deleted-session> --no-restore-pruned`
- [ ] **Expected:** name resolves, pruned detection fires ("not resuming" + hint), exit 1 -- resolution and the pruned pipeline compose.
